Constraint Model (Stage-Gate) vs. Outcome Model (PA-WBS) | March 2026

This document maps the 6 in-scope stages (S0-S5) of the constraint model to the 11 outcomes (PA-OC-01 through PA-OC-11) of the outcome model. Stages S6-S8 (application code work) are out of scope for this engagement. The goal is to determine whether both models can coexist -- the team delivers on outcomes while the PM reports on constraint stages -- or whether the mapping is too complex to maintain.

Key Findings

Strong Alignment (5 of 6 in-scope stages)
S0 through S3 and S5 all map cleanly to individual outcomes. The scope, work packages, decisions, and exit criteria are nearly identical. Reporting on one translates directly to the other.
S4 is the Only In-Scope Complication
S4 (Config Pipeline Operational) maps to 4 separate outcomes (PA-OC-05 through PA-OC-08). In the constraint model, S4 is "done" when all 4 work packages are done. In the outcome model, PA-OC-05/06/07 can complete independently, and PA-OC-08 (reliability) is iterative. Status rollup requires aggregation.
S6-S8 Out of Scope
Stages S6 (App Code in VCS), S7 (App CI/CD), and S8 (Dev Adoption) are out of scope for this engagement. Application code work is not included in the current Pipeline Automation project. Two new outcomes (PA-OC-10 Fallback and PA-OC-11 Pattern Replication) address conditional scenarios the constraint model does not explicitly cover.

Reporting Translation: How it Works in Practice

Bottom line: The mapping is clean enough that both models can coexist. The PM reports on constraint stages using status data from outcome deliverables. One complication exists (S4), and it is manageable.

Deep Dive: The S4 Split

S4 is the only stage that requires translation effort. Here is exactly how the 4 outcomes roll up:

S4 Work Package Outcome Can complete independently? S4 reporting impact
WP 4.1: Build Pipeline PA-OC-05 YES When PA-OC-05 is done, report S4 WP 4.1 as complete
WP 4.2: Deploy Pipeline PA-OC-06 YES (after PA-OC-05) When PA-OC-06 is done, report S4 WP 4.2 as complete
WP 4.3: Drift Detection PA-OC-07 YES (parallel with PA-OC-05/06) When PA-OC-07 is done, report S4 WP 4.3 as complete. This may complete weeks before WP 4.1/4.2.
WP 4.4: Reliability PA-OC-08 NO (needs PA-OC-05+06+07) When PA-OC-08 is done, report S4 WP 4.4 as complete. S4 stage = done.
The S4 reporting advantage: In the constraint model, S4 looks like one block from Week 10-14. In reality, drift detection (PA-OC-07) can start at Week 7 and complete by Week 11, while build/deploy work continues. The outcome model shows this progress; the constraint model hides it. When reporting S4 to leadership, you can show work-package-level progress that the outcome model makes visible.
