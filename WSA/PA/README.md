# Pipeline Automation (PA)

Normative planning for the **Pipeline Automation** capability (NGINX config pipeline, Jira root **WSA-2656**).

## Canonical documents

| Artifact | Purpose |
|----------|---------|
| **`PA-WBS.md`** | Outcome-based **Work Breakdown Structure** — success criteria, deliverables, dependencies, risks, decisions, Jira mapping (human-readable). |
| **`pa-outcomes.json`** | Optional **structured** outcome IDs ↔ Jira Epic keys for scripts and reconciliation; **keep in sync** with `PA-WBS.md` §10 when epics change. |
| **`PA-Outcome-map.html`** | Visual timeline and outcome bands. |
| **`PA-kanban.html`** | Per-outcome boards; **Jira-backed** via embedded export snapshot. |
| **`PA-Constraint-vs-Outcome-Map.html`** | Stage-gate vs outcome model comparison. |

## Jira exports (`Jira/`)

- **Dated full exports:** `PA-Jira-MM-DD-YYYY-json.json` — **keep multiple** dated files for history; each live export run adds or overwrites the dated file for that day per `Scripts/jira-export-pa.js`.
- **Kanban snapshot (current):** `pa-kanban-jira-status.json` and `pa-kanban-jira-status.js` — regenerated by the export script (or `Scripts/jira-kanban-status-from-export.js` from a saved export).
- See **`Jira/README.md`** for field shapes and import JSON (`../Output/PA-WBS-Jira-Import.json`).

**Refresh from Jira (project root):**

```bash
node Scripts/jira-export-pa.js
```

Requires `JIRA_URL`, `JIRA_USERNAME`, `JIRA_API_TOKEN` (e.g. in `.cursor/.env`).

## Folder layout

| Path | Role |
|------|------|
| **`Input/`** | Stakeholder inputs for WBS loads; processed files move to **`Input/Archive/<date>/`**. |
| **`Output/`** | Target-state Jira import JSON (`PA-WBS-Jira-Import.json`) and **`Output/Archive/`** snapshots. |
| **`Update-Reports/`** | WBS load reports (`WBS-Load-mm-dd-yyyy.md`). |
| **`Archive/`** | Point-in-time copies of **`PA-WBS.md`** before major loads (`PA-WBS-mm-dd-yyyy.md`). |

## Conventions

- Outcome IDs: **PA-OC-00** … **PA-OC-11**; deliverables **PA-OC-XX.Y**.
- **Jira** is the authoritative backlog for execution; **PA-WBS.md** is normative for outcomes and IDs.
- Cursor rules: **`.cursor/rules/pa.mdc`**; WBS load / Jira alignment skill: **`.cursor/skills/wbs-update-pattern/`**.

**Sibling capabilities:** VI and WM use the same conventions — **`VI-WBS.md`** / **`WM-WBS.md`**, **`VI-WBS-Jira-Import.json`** / **`WM-WBS-Jira-Import.json`**, dated **`VI-Jira-*`** / **`WM-Jira-*`** under each **`Jira/`** folder. See **`VI/README.md`** and **`WM/README.md`**.
