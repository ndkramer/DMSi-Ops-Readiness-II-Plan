What Changes from the Stage-Gate Model

Process Before Tool

WM-OC-01 and WM-OC-02 establish the operating model before any tool is selected. WM-OC-03 treats tool selection as a fit-to-process exercise -- the operating agreement defines the requirements, and the tool is evaluated against them. The tool serves the workflow, not the other way around.

Parallel Execution Unlocked

WM-OC-04 (Queue Disposition) runs alongside WM-OC-05 (Initial Adoption) instead of waiting. WM-OC-08 (Scorecard) can begin design during WM-OC-06 with partial data, using a "should" dependency on WM-OC-07 instead of a hard gate. This recovers 2-4 weeks of serial delay.

Iterative Validation Made Explicit

WM-OC-05, WM-OC-06, and WM-OC-07 each have defined validation periods (3-4 weeks), max iteration cycles (2-3), and explicit fallback actions. Instead of repeating a failed stage, the team loops within the outcome with a clear policy for when to escalate.
