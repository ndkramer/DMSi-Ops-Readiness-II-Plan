What Changes from the Stage-Gate Model

Parallel Execution Unlocked

PA-OC-07 (Drift Detection) can start as soon as configs are in VCS (PA-OC-04), running in parallel with PA-OC-05 (Build Pipeline) and PA-OC-06 (Deploy Pipeline). At 1.5 FTEs, parallelism is constrained but PA-OC-05 + PA-OC-07 overlap saves ~3 weeks vs. fully sequential.

Iterative Validation Explicit

PA-OC-08 (Pipeline Reliability) has a 2-week validation window with max 3 iterations. If reliability stays below 90%, the team loops back within the same outcome. Fallback triggers PA-OC-10 instead of an unplanned failure response.

Pipeline Replication Planned

Additional config pipeline work (PA-OC-11) is planned but not scheduled. It only activates after the NGINX POC validates the framework and leadership decides to extend the pattern to other config types (PASOE, customer runtime, infrastructure). No phantom blockers from future scope.
