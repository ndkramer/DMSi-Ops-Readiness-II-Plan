How do I get a GitHub token for this report?

There is no separate “project token.” This page uses the GitHub API as you. Create a Personal Access Token under your GitHub login with permission to read the organization project and the repositories whose issues show up on that project.

Fine-grained token (recommended)

  1. Open GitHub → Settings → Developer settings → Fine-grained tokens → Generate new token.
  2. Under Resource owner, select your organization (e.g. DynamoLLC-Hub) if the project is org-owned.
  3. Repository access: include every repo that contributes issues to the project, or “All repositories” if your org allows it.
  4. Permissions: Repository → Issues (Read), Metadata (Read). Organization → Projects (Read) (read-only).
  5. Generate the token and copy it once; paste it via Set GitHub token (stored in this browser until you click Forget saved token), or add a Stalled-Blocked-github-token.local.json next to this HTML on the server (see the .example file in the repo; do not commit the real file). Or use ?github_token=… only in trusted environments.

Classic token (alternative)

  1. Generate new token (classic).
  2. Enable scopes: read:project (or full project for classic boards), read:org, and repo if issues are in private repositories.

If you see “Project #1 not found” or similar: the board may be a classic project (this page now falls back to that API) or your token may lack Organization → Projects (Read). If the org uses SAML SSO, open the token in GitHub → Configure SSO and authorize it for DynamoLLC-Hub.

If you see “Failed to fetch” or 403: embedders may block api.github.com, or Lambda may reject requests without the gate token. Use both: ?token=… (same value as the Capability Map / Confluence iframe) and &github_token=… (GitHub PAT). Force proxy with &github_proxy=1 if needed.

Confluence iframe: sandboxed frames can make location.origin the string null (this page uses location.href for the Lambda host). Those frames also treat calls to your Lambda as cross-origin, so the Lambda must return CORS headers on /github-proxy and /github-api — deploy the latest index.mjs. Prefer Set GitHub token over putting a PAT in the URL; revoke any leaked PAT.
